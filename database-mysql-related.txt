১. টার্মিনাল/কমান্ড লাইন কমান্ড
এই কমান্ডগুলো ডাটাবেস সার্ভারের সাথে সংযোগ স্থাপন, ব্যাকআপ তৈরি এবং ইম্পোর্ট করার জন্য ব্যবহৃত হয়। এগুলি চালানোর সময় আপনার MySQL এর bin ফোল্ডারের Path সেট করা থাকতে হবে।

কমান্ড	কাজ	উদাহরণ
লগইন (Login)	MySQL সার্ভারে প্রবেশ করে SQL কোয়েরি চালানোর জন্য।	mysql -u root -p
এক্সপোর্ট/ব্যাকআপ	একটি ডাটাবেসকে .sql ফাইলে ব্যাকআপ বা এক্সপোর্ট করা।	mysqldump -u root -p DB_NAME > backup.sql
ইম্পোর্ট/রিস্টোর	একটি .sql ফাইল থেকে ডাটাবেসে ডেটা ইম্পোর্ট করা।	mysql -u root -p DB_NAME < backup.sql
ভার্সন চেক	আপনার ইনস্টল করা MySQL সার্ভারের ভার্সন জানা।	mysql --version

২. SQL কমান্ড (MySQL ক্লায়েন্টের ভেতরে ব্যবহারের জন্য)
একবার mysql -u user -p কমান্ড দিয়ে লগইন করার পর আপনি নিচের কমান্ডগুলি ব্যবহার করবেন:

ক. ডাটাবেস পরিচালনা (Database Management)
কমান্ড	কাজ	উদাহরণ
ডাটাবেস তৈরি	একটি নতুন ডাটাবেস তৈরি করে।	CREATE DATABASE new_database;
ডাটাবেস দেখা	সার্ভারে উপলব্ধ সব ডাটাবেসের তালিকা দেখায়।	SHOW DATABASES;
ডাটাবেস নির্বাচন	কোন ডাটাবেসের টেবিল/ডেটা নিয়ে কাজ করতে চান, তা নির্ধারণ করা।	USE database_name;
ডাটাবেস মোছা	ডাটাবেসটি স্থায়ীভাবে মুছে দেয়। (সাবধানতা অবলম্বন করুন!)	DROP DATABASE database_name;

খ. টেবিল পরিচালনা (Table Management)
কমান্ড	কাজ	উদাহরণ
টেবিল তৈরি	একটি নতুন টেবিল এবং তার কলামগুলি তৈরি করে।	CREATE TABLE users (id INT, name VARCHAR(100));
টেবিল দেখা	বর্তমানে নির্বাচিত ডাটাবেসের সব টেবিলের তালিকা দেখায়।	SHOW TABLES;
টেবিলের গঠন দেখা	একটি নির্দিষ্ট টেবিলের কলাম, ডেটাটাইপ ইত্যাদি দেখায়।	DESCRIBE users;
টেবিল মোছা	টেবিলটি স্থায়ীভাবে মুছে দেয়।	DROP TABLE users;
টেবিল পরিবর্তন	টেবিলে নতুন কলাম যোগ, মোছা বা পরিবর্তন করা।	ALTER TABLE users ADD email VARCHAR(100);

গ. ডেটা ম্যানিপুলেশন (Data Manipulation - DML)
এই কমান্ডগুলো টেবিলের ভেতরের ডেটার সাথে সম্পর্কিত।

কমান্ড	কাজ	উদাহরণ
ডেটা নির্বাচন	টেবিল থেকে ডেটা খুঁজে বের করা।	SELECT name, email FROM users WHERE id = 1;
ডেটা ঢোকানো	টেবিলে নতুন সারি (Row) যোগ করা।	INSERT INTO users (name, email) VALUES ('Moinul', 'm@example.com');
ডেটা আপডেট	টেবিলের বিদ্যমান ডেটা পরিবর্তন করা।	UPDATE users SET email = 'new@mail.com' WHERE name = 'Moinul';
ডেটা মোছা	টেবিল থেকে ডেটা মুছে দেওয়া।	DELETE FROM users WHERE id = 1;

ঘ. লেনদেন (Transaction Control - TCC)
এই কমান্ডগুলি ডেটাবেসে একাধিক DML অপারেশনকে এক ইউনিট হিসেবে নিয়ন্ত্রণ করতে ব্যবহৃত হয়।

কমান্ড	কাজ	উদাহরণ
লেনদেন শুরু	একটি লেনদেন শুরু করা।	START TRANSACTION;
পরিবর্তন সংরক্ষণ	লেনদেনের সব পরিবর্তন স্থায়ীভাবে সেভ করা।	COMMIT;
পরিবর্তন বাতিল	লেনদেনের সব পরিবর্তন বাতিল করে আগের অবস্থায় ফিরে যাওয়া।	ROLLBACK;

ঙ. ইউজার ও পারমিশন (User & Privilege Management)
কমান্ড	কাজ	উদাহরণ
ইউজার তৈরি	একটি নতুন ডাটাবেস ইউজার তৈরি করা।	CREATE USER 'newuser'@'localhost' IDENTIFIED BY 'password';
পারমিশন প্রদান	ইউজারকে ডাটাবেস বা টেবিল ব্যবহারের অনুমতি দেওয়া।	GRANT ALL PRIVILEGES ON db_name.* TO 'newuser'@'localhost';
পারমিশন রিফ্রেশ	নতুন পারমিশন কার্যকর করার জন্য।	FLUSH PRIVILEGES;
